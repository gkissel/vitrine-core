# PWABuilder Packaging Guide

This project is configured as a Progressive Web App and can be packaged for Android and iOS using PWABuilder.

Production PWA URL:

```text
https://app.ervamateparaobrasil.com.br
```

Backend URL:

```text
https://medusa.ervamateparaobrasil.com.br
```

## What PWABuilder Needs

PWABuilder analyzes the deployed storefront URL. The app must be available over HTTPS and expose the PWA metadata from the production domain.

Current PWA configuration lives in:

- `apps/front-end/app/manifest.ts` — Web App Manifest generated by Next.js.
- `apps/front-end/public/sw.js` — service worker.
- `apps/front-end/public/offline.html` — offline fallback page.
- `apps/front-end/components/pwa-service-worker-registrar.tsx` — service worker registration.
- `apps/front-end/lib/metadata.ts` — root metadata, manifest link, icons, Apple web app metadata.
- `apps/front-end/next.config.ts` — standalone Docker output and PWA-related headers.
- `apps/front-end/public/icons/` — app icons.
- `apps/front-end/public/images/pwa-screenshot-wide.png` — wide screenshot.
- `apps/front-end/public/images/pwa-screenshot-mobile.png` — mobile screenshot.
- `apps/front-end/public/browserconfig.xml` — Microsoft tile metadata.

## PWA Checklist

Before generating packages, confirm:

- The storefront is deployed at `https://app.ervamateparaobrasil.com.br`.
- The page loads without certificate warnings.
- `https://app.ervamateparaobrasil.com.br/manifest.webmanifest` returns the manifest.
- The manifest has:
  - `name` or `short_name`.
  - `start_url`.
  - `scope`.
  - `display` or `display_override`.
  - `prefer_related_applications: false`.
  - 192x192 and 512x512 icons.
  - maskable icons.
  - screenshots for install prompts/store metadata.
- `https://app.ervamateparaobrasil.com.br/sw.js` is reachable.
- The service worker registers successfully in the browser.
- The app has an offline fallback through `offline.html`.
- The app works on a secure origin: HTTPS, localhost, or loopback.

## Generate Android Package

1. Open:

   ```text
   https://www.pwabuilder.com/
   ```

2. Enter:

   ```text
   https://app.ervamateparaobrasil.com.br
   ```

3. Review the score and fix any warnings that are relevant.
4. Choose the Android platform package.
5. Download the generated ZIP.
6. Extract the ZIP into:

   ```text
   android/
   ```

7. Do not commit signing credentials or keystores.

## Generate iOS Package

1. Open:

   ```text
   https://www.pwabuilder.com/
   ```

2. Enter:

   ```text
   https://app.ervamateparaobrasil.com.br
   ```

3. Review the score and fix any warnings that are relevant.
4. Choose the iOS platform package.

6. Extract the ZIP into:

   ```text

   ```

7. Do not commit Apple certificates, provisioning profiles, App Store Connect API keys, or private keys.

## Important Environment Values

The storefront build should use the production backend URL:

```env
MEDUSA_BACKEND_URL=https://medusa.ervamateparaobrasil.com.br
NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY=YOUR_MEDUSA_PUBLISHABLE_KEY
```

Because `NEXT_PUBLIC_*` values are inlined by Next.js, set them as build args in Dokploy as well as runtime environment variables.

## Android/iOS Folder Policy

The repository has two prepared folders populated from the PWABuilder downloads:

- `android/` — Android package, APK/AAB releases, `assetlinks.json`, and generated Gradle source.
- `ios/` — iOS package and generated Xcode source.

Secrets must stay outside git. Keystores, Apple certificates, provisioning profiles, private keys, and passwords must not be committed.

## Android Digital Asset Links

PWABuilder generated the Android Digital Asset Links file at:

```text
android/assetlinks.json
```

It was also copied to the storefront public folder so it is served at the required URL:

```text
apps/front-end/public/.well-known/assetlinks.json
https://app.ervamateparaobrasil.com.br/.well-known/assetlinks.json
```

The Android package declared there is:

```text
br.com.ervamateparaobrasil.app.twa
```

## After Downloading New Packages

If packages are regenerated in PWABuilder, replace the files in `android/` and `ios/`, then confirm that no signing secrets were copied.

Then build/sign them using the platform tooling:

- Android: Android Studio, Gradle, and Google Play Console.
- iOS: Xcode and App Store Connect.

If a package includes generated metadata that points to the wrong domain, regenerate the package after redeploying the PWA with the correct production URL.
